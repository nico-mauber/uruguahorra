1. VISIÓN GENERAL DEL PROYECTO
================================================================================

OBJETIVO PRINCIPAL:
Aplicación móvil que ayuda a personas en Latinoamérica a empezar y mantener el hábito 
de ahorrar e iniciarse en inversiones de forma simple y divertida, usando gamificación 
para reforzar buenos hábitos financieros.

PROBLEMA A RESOLVER:
En LATAM muchas personas no ahorran ni invierten por tres motivos:
1. Lo perciben como complicado
2. No saben por dónde empezar  
3. No logran mantener la constancia

PROPUESTA DE VALOR:
- Para usuarios: Claridad, motivación diaria y planes simples para ahorrar e iniciarse 
  en inversiones sin miedo
- Para empresas (futuro B2B): Bienestar financiero para empleados como beneficio corporativo

USUARIOS OBJETIVO:
- Jóvenes 18-30 que quieren empezar a ahorrar
- Empleados 25-40 con poco tiempo que desean automatizar hábitos
- Freelancers que buscan ordenarse y crear un colchón de emergencia

2. CARACTERÍSTICAS DEL MVP (4-6 SEMANAS)
================================================================================

QUÉ INCLUYE:
✓ Onboarding por metas (colchón de emergencia / viaje / deudas / compras)
✓ Microaportes manuales ($1/$2/$5) y "redondeo simulado" sobre compras registradas
✓ Retos semanales gamificados y sistema de rachas
✓ Ranking entre amigos y grupos de ahorro (squads)
✓ Educación en cápsulas cortas (60-90 segundos) en español
✓ Importación de movimientos por CSV (sin scraping ni custodia)
✓ Paywall simple con modelo freemium → premium (mensual/anual)
✓ Notificaciones push para recordatorios de rachas y metas

QUÉ NO INCLUYE EL MVP:
✗ Custodia de dinero real
✗ Ejecución de inversiones
✗ Asesoría financiera personalizada
✗ Integración bancaria directa

MODELO DE DATOS (13 TABLAS PRINCIPALES):
1. users - Usuarios del sistema (email, país, moneda, premium, fechas)
2. goals - Metas por usuario (monto objetivo, fecha objetivo, progreso)
3. micro_contributions - Registros de microaportes (monto, fuente: manual/roundup)
4. challenges - Catálogo de retos disponibles
5. user_challenges - Progreso de retos por usuario
6. squads - Grupos/equipos de ahorro
7. squad_members - Miembros de cada squad
8. transactions_raw - Datos importados de CSV para análisis
9. learnings - Cápsulas educativas
10. user_progress - Consumo de contenidos educativos
11. subscriptions - Estado de suscripción (Stripe/MercadoPago)
12. paywall_events - Analítica del paywall
13. audit_logs - Trazabilidad y auditoría

API ENDPOINTS PRINCIPALES:
- POST /auth/login - Login con email OTP (magic link)
- GET/POST /goals - Crear y listar metas
- POST /contributions - Registrar microaporte
- GET /challenges - Catálogo de retos
- POST /challenges/{id}/complete - Completar reto
- GET /leaderboard?squad_id= - Ranking
- POST /transactions/upload-csv - Subir y parsear CSV
- GET /reports/summary - Progreso del mes
- POST /billing/checkout - Iniciar suscripción
- POST /billing/webhooks - Recibir eventos de pago

4. SISTEMA DE GAMIFICACIÓN
================================================================================

MECÁNICAS DE XP:
- Aporte: +2 XP por cada $1 (hasta 10 XP por evento)
- Completar reto: +30 XP
- Racha diaria: +5 XP
- Niveles: nivel = floor(sqrt(totalXP)/2)

SISTEMA DE RACHAS:
- Se rompe si pasan 48h sin aporte o reto
- 1 protección por mes disponible
- Notificaciones de recordatorio a las 24h

QUESTS Y RETOS:
- Colecciones semanales de retos
- Tipos: ahorro, educación, social
- Puntos variables según dificultad

================================================================================
5. SEGURIDAD Y CUMPLIMIENTO
================================================================================

MEDIDAS DE SEGURIDAD:
✓ Autenticación con email/OTP y 2FA opcional
✓ Cifrado en tránsito (HTTPS) y datos sensibles en reposo
✓ RLS (Row Level Security) en todas las tablas por user_id
✓ Políticas de acceso mínimo y roles de servicio
✓ Encriptación de datos sensibles (IPs con hash + salt)

CUMPLIMIENTO LEGAL:
✓ Mensaje legal: "Contenido educativo; no constituye asesoría financiera"
✓ Protección de datos: consentimiento, borrado, exportación
✓ Sin custodia de dinero en el MVP (solo educación y hábitos)
✓ Integración futura solo con partners regulados
