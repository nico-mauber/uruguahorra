# UX Design System Prompt - Uruguahorra
## Principios Psicológicos para Interfaces Financieras

### 🧠 PSICOLOGÍA DEL COMPORTAMIENTO FINANCIERO

#### 1. Loss Aversion (Aversión a la Pérdida)
- **Color Psychology**: Rojo para gastos (activar precaución), Verde para ahorros (refuerzo positivo)
- **Visual Weight**: Gastos con más prominencia visual que ingresos (efecto cognitivo)
- **Feedback Timing**: Mostrar impacto inmediato de cada transacción en objetivos

#### 2. Mental Accounting (Contabilidad Mental)
- **Categorización Visual**: Emojis + colores únicos por categoría
- **Separación Clara**: Diferentes "cubetas" mentales (necesidades vs gustos)
- **Budget Buckets**: Visualizar como recipientes que se llenan/vacían

#### 3. Present Bias (Sesgo del Presente)
- **Gratificación Inmediata**: Animaciones de éxito instantáneas
- **Future Visualization**: Mostrar progreso hacia objetivos futuros como presente
- **Streaks**: Sistema de rachas para mantener hábitos

#### 4. Social Proof (Prueba Social)
- **Comparaciones Anónimas**: "Usuario promedio gasta X en cafés"
- **Squad Dynamics**: Mostrar comportamientos grupales positivos
- **Achievement Sharing**: Celebraciones públicas de logros

### 🎨 DESIGN PRINCIPLES

#### Color Palette Psicológica
```
Gastos (Loss Aversion):
- Rojo Principal: #FF6B6B (alerta suave, no agresivo)
- Rojo Secundario: #FF8E8E (transiciones)
- Rojo Background: #FFF5F5 (contexto suave)

Ahorros (Positive Reinforcement):
- Verde Principal: #51CF66 (crecimiento, vida)
- Verde Secundario: #8CE99A (celebración)
- Verde Background: #F3FFF3 (ambiente positivo)

Neutrales (Balance):
- Azul Principal: #339AF0 (confianza, estabilidad)
- Gris Inteligente: #6C757D (información sin emoción)
- Blanco Puro: #FFFFFF (claridad mental)
```

#### Typography Hierarchy (Jerarquía Cognitiva)
```
Money Amounts (Máxima Atención):
- Font: 24-32px, Weight: 700
- Color: Contextual (rojo/verde)
- Shadow: Sutil para profundidad

Categorías (Reconocimiento Rápido):
- Font: 16-18px, Weight: 600
- Emoji: 20px (reconocimiento icónico)

Descriptions (Lectura Fluida):
- Font: 14-16px, Weight: 400
- Color: Gris inteligente
- Line Height: 1.4 (legibilidad)
```

### 🎯 INTERACTION PATTERNS

#### 1. Quick Entry (3-Tap Rule)
```
Tap 1: FAB → Modal abierto
Tap 2: Cantidad (teclado numérico grande)
Tap 3: Categoría (visual, emojis grandes)
[Auto-save] → Feedback inmediato
```

#### 2. Swipe Actions (Muscle Memory)
```
Swipe Right: Editar (verde suave)
Swipe Left: Eliminar (rojo suave)
Long Press: Detalles completos
```

#### 3. Progressive Disclosure
```
Nivel 1: Cantidad + Categoría (esencial)
Nivel 2: Descripción + Fecha (tap para expandir)
Nivel 3: Etiquetas + Notas (power users)
```

### 🏆 GAMIFICATION PSYCHOLOGY

#### Achievement Types
```
Frequency Based:
- "Racha de 7 días registrando gastos" 🔥
- "100 transacciones registradas" 📊

Behavior Based:
- "Semana sin gastos innecesarios" 💎
- "Presupuesto cumplido 3 meses" 🎯

Social Based:
- "Top 3 en tu squad este mes" 👑
- "Ayudaste a 5 amigos con sus objetivos" 🤝
```

#### Progress Visualization
```
Linear Progress: Para objetivos concretos
Circular Progress: Para hábitos/streaks
Stacked Progress: Para categorías múltiples
Animated Counters: Para celebrar logros
```

### 📱 MOBILE-FIRST UX PATTERNS

#### Touch Targets
```
Minimum: 44x44px (Apple HIG)
Preferred: 48x48px (Material Design)
Critical Actions: 56x56px (easy thumb reach)
```

#### Thumb-Friendly Layout
```
Primary Actions: Bottom 1/3 de pantalla
Secondary Actions: Middle 1/3
Information Display: Top 1/3
FAB Position: Bottom right, 16px margin
```

#### Loading States (Perceived Performance)
```
Skeleton Screens: Para listas largas
Optimistic Updates: Para acciones inmediatas
Progress Indicators: Para procesos conocidos
Shimmer Effects: Para contenido dinámico
```

### 🎬 ANIMATION GUIDELINES

#### Emotional Animations
```
Success (Dopamine Trigger):
- Confetti burst: 800ms
- Scale bounce: 150ms ease-out
- Green glow: 300ms fade-in

Warning (Attention):
- Shake animation: 200ms x3
- Red pulse: 500ms
- Scale emphasis: 100ms

Neutral (Fluidity):
- Slide transitions: 250ms ease-in-out
- Fade transitions: 200ms
- Scale micro-interactions: 100ms
```

#### Performance Targets
```
60 FPS: Todas las animaciones críticas
120ms: Tiempo máximo para feedback táctil
250ms: Transiciones entre pantallas
16ms: Frame time para animaciones fluidas
```

### 🔊 SOUND DESIGN (Accessibility)

#### Audio Cues
```
Success: Suave "ding" (440Hz, 100ms)
Warning: Doble beep (880Hz, 50ms x2)
Button Tap: Click sutil (200Hz, 20ms)
Achievement: Celebración musical (3 notas ascendentes)
```

### 🌓 DARK MODE ADAPTATIONS

#### Color Adjustments
```
Backgrounds: Negro verdadero → Gris oscuro (#1A1A1A)
Text: Blanco puro → Gris claro (#E0E0E0)
Accent Colors: Mantener pero con 20% menos saturación
Shadows: Reemplazar con bordes sutiles
```

### 📊 DATA VISUALIZATION PSYCHOLOGY

#### Chart Types por Contexto
```
Line Charts: Tendencias temporales (progreso/regresión)
Bar Charts: Comparaciones categóricas (gastos por tipo)
Pie Charts: Composición total (distribución presupuesto)
Area Charts: Volumen acumulativo (ahorros creciendo)
```

#### Information Hierarchy
```
Most Important: Centro, grande, colorido
Important: Prominente, buen contraste
Context: Sutil, gris, pequeño
Metadata: Muy sutil, solo si es necesario
```

### 🎪 MICRO-INTERACTIONS CATALOG

#### Button States
```
Idle: Base state, sutil shadow
Hover: Lift effect (2px elevation)
Active: Press down (-1px), color darken 10%
Disabled: 40% opacity, no interactions
```

#### Form Feedback
```
Valid Input: Verde sutil border (1px)
Invalid: Rojo border + shake + message
Loading: Pulse animation en botón
Success: Checkmark animation (500ms)
```

#### List Interactions
```
Pull to Refresh: Spinner + "Actualizando..."
Empty State: Ilustración + mensaje motivacional
No Internet: Retry button + cached data notice
Error State: Clear error message + retry action
```

### 🧪 A/B TESTING FRAMEWORK

#### Key Metrics
```
Engagement:
- Transactions logged per day
- Time spent in app
- Feature discovery rate

Satisfaction:
- Task completion rate
- Error recovery rate
- User retention week-over-week
```

#### Test Categories
```
Visual: Colors, fonts, spacing
Interaction: Gestures, navigation, flows
Content: Messages, copy, imagery
Structure: Information architecture, grouping
```

### 🌍 ACCESSIBILITY (WCAG 2.1 AA)

#### Color Contrast
```
Normal Text: 4.5:1 minimum
Large Text: 3:1 minimum
Interactive Elements: 3:1 minimum
Focus Indicators: High contrast borders
```

#### Screen Reader Support
```
All images: Alt text descriptivo
Interactive elements: Accessible labels
State changes: Announcements claras
Navigation: Skip links, landmarks
```

#### Motor Accessibility
```
Touch targets: 44px minimum
Gesture alternatives: Always provide
Time limits: Adjustable or none
Motion sensitivity: Respect system preferences
```

### 📋 COMPONENT CHECKLIST

#### Antes de implementar cualquier componente:
- [ ] ¿Reduce la fricción cognitiva?
- [ ] ¿Proporciona feedback inmediato?
- [ ] ¿Es accesible para todos los usuarios?
- [ ] ¿Respeta los patrones de la plataforma?
- [ ] ¿Funciona bien en ambos temas (light/dark)?
- [ ] ¿Las animaciones mejoran la UX o son decorativas?
- [ ] ¿El rendimiento es óptimo en dispositivos de gama baja?

### 🎨 COMPONENT DESIGN PATTERNS

#### Modal Design
```
Entry Animation: Slide up from bottom (300ms)
Backdrop: Semi-transparent black (0.5 opacity)
Close Actions: Swipe down, tap outside, X button
Content: Max 80% screen height, scroll if needed
```

#### Card Design
```
Elevation: 2px shadow para separación
Radius: 12px para suavidad moderna
Padding: 16px interno consistente
Spacing: 12px entre cards
```
